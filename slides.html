Part 1

Remote Sensing Foundations

Landsat ถ่ายอะไรบ้าง?

ดาวเทียม Landsat บันทึกความสว่างใน 6 ย่านคลื่นหลัก (ตาคนเห็นแค่ 3 ย่าน)

Blue
0.45 μm
Green
0.56 μm
Red
0.66 μm
NIR
0.86 μm
SWIR-1
1.6 μm
SWIR-2
2.2 μm

Blue, Green, Red = สีที่ตามองเห็น (RGB)
NIR (Near Infrared) = ตามองไม่เห็น · พืชสะท้อน NIR มาก (กุญแจของการแยกพืช)
SWIR (Short-Wave Infrared) = บอกความชื้น + วัตถุก่อสร้าง

μm = micrometer (10⁻⁶ เมตร) หน่วยวัดคลื่นแสง

🤔 Poll: คุณคิดว่า...

ในภาพดาวเทียม NIR (Near Infrared) —
วัตถุไหนสะท้อนมากที่สุด?
A. น้ำทะเล
B. ป่าไม้สุขภาพดี
C. ถนนคอนกรีต
D. ทรายทะเล
คำตอบ: B — ป่าไม้สุขภาพดี 🌲
Chlorophyll ในใบไม้ดูดซับ Red แต่สะท้อน NIR มาก (ธรรมชาติทำให้ใบไม่ร้อน)
→ นี่คือเหตุผลที่ NDVI = (NIR - Red) / (NIR + Red) วัดพืชได้ดี

Spectral Signatures — "ลายนิ้วมือ" ของวัตถุ

แต่ละวัตถุสะท้อนแสงใน 6 bands ด้วย pattern เฉพาะ:

🌲 พืช: Red ต่ำ → NIR สูงมาก (chlorophyll) → SWIR ปานกลาง
🏙️ เมือง: ค่อนข้างสูงทุก band โดยเฉพาะ SWIR (วัตถุก่อสร้าง)
💧 น้ำ: Blue สูง → NIR/SWIR ต่ำมาก (น้ำดูดซับ)
🌾 เกษตร: คล้ายพืชแต่ขึ้นกับช่วงเวลา (เก็บเกี่ยวแล้วจะต่ำ)
อุปมา: เหมือน ECG ของคน — แต่ละคนมี pattern หัวใจต่างกัน Spectral signature = "ECG" ของวัตถุ

Spectral Indices — วัดเฉพาะที่สนใจ

แทนใช้ทุก band เราเอา 2 bands มา normalize → ค่าอยู่ -1 ถึง +1

Indexสูตรใช้แยกค่าสูง =
NDVI(NIR - Red) / (NIR + Red)พืชพืชสุขภาพดี
NDWI(Green - NIR) / (Green + NIR)น้ำแหล่งน้ำ
NDBI(SWIR - NIR) / (SWIR + NIR)เมืองอาคาร/ถนน
ทำไมต้อง Normalize?
เพราะค่าสัมบูรณ์เปลี่ยนตามแสงแดด · Normalize แล้วเทียบข้าม scene ได้
Threshold rule of thumb: NDVI > 0.3 = พืช · NDWI > 0 = น้ำ · NDBI > -0.05 = เมือง

Part 2

Urban Expansion Analysis

LULC Classification — แยก 8 ประเภทที่ดิน

Index 3 ตัวเห็นภาพคร่าว ๆ แต่ถ้าอยาก แยกได้ละเอียดกว่า ต้องใช้ ML

🌲 Vegetation 🌿 Shrubland 🌾 Grassland 🌾 Cropland
🏙️ Built-up 🏜️ Bareland 💧 Water body ❔ Unclassified
Workflow:
  1. Training points (GeoJSON) — human label แล้ว ~หลายร้อยจุด
  2. ดึงค่า 6 bands ที่แต่ละจุด → X (features)
  3. Train Random Forest (X → class)
  4. Apply model ทุก pixel ในภาพ → LULC map

Random Forest Intuition

Random Forest = ป่าของ Decision Trees — แต่ละต้น vote แล้วเอา majority

อุปมา: เหมือน panel ผู้เชี่ยวชาญ 100 คน
แต่ละคนเห็นข้อมูลแค่บางส่วน · vote → คำตอบของกลุ่ม
→ แม่นกว่าถามคนเดียว (ลด overfitting) 
from sklearn.ensemble import RandomForestClassifier

model = RandomForestClassifier(
    n_estimators=100,   # จำนวนต้นไม้
    max_depth=20,       # ลึกสุดของแต่ละต้น
    n_jobs=-1,          # ใช้ CPU ทุก core
)
model.fit(X_train, y_train)

accuracy = model.score(X_test, y_test)
n_estimators = จำนวนต้นไม้ (ยิ่งเยอะยิ่งดี แต่ช้าขึ้น · 100 โอเคสำหรับส่วนใหญ่)
max_depth = ความลึกของ tree (มากไป = overfit)

🧠 เดาก่อน — เราจะเห็นคำตอบจริงตอนรัน notebook

วัดการเปลี่ยนแปลง (pixel → km²)

Landsat = 30m/pixel → 1 pixel = 900 m² = 0.0009 km²

def compute_area(lulc_map, pixel_area=900):
    unique, counts = np.unique(lulc_map, return_counts=True)
    return pd.DataFrame({
        'class': unique,
        'area_km2': counts * pixel_area / 1_000_000
    })

area_2015 = compute_area(lulc_2015)
area_2025 = compute_area(lulc_2025)
change = ((area_2025 - area_2015) / area_2015) * 100
📊 ผลที่คาด:
  • Built-up มักจะเพิ่ม (urbanization)
  • Cropland มักจะลด (ถูก convert เป็นเมือง)
  • Water body มักจะคงที่ (อาจเปลี่ยนตามฤดู/ภัยแล้ง)

🔮 CA-Markov: จำลองอนาคต

ถ้ารู้ pattern การเปลี่ยนแปลง ในอดีต → จำลองต่อไปในอนาคตได้

Markov Chain

P(next_class | current_class)
คำนวณจาก 2015→2025 transition

Cellular Automata

แต่ละ pixel ดูเพื่อนบ้าน →
pixel ถัดไปเป็นเมือง ก็มีโอกาสเป็นเมืองตาม

อุปมา (econ): เหมือนโมเดล network effect
· Markov = trend โดยรวม (เมืองขยาย)
· CA = spatial clustering (เมืองเติบโตติดเมือง ไม่ใช่กลางนา)

Transition Matrix (ตัวอย่าง)

จาก → ไปVegetationCroplandBuilt-upWater
Vegetation0.850.080.050.02
Cropland0.150.700.130.02
Built-up0.020.010.970.00
Water0.010.010.000.98

อ่านยังไง: แถวคือ class ปัจจุบัน · คอลัมน์คือ class ถัดไป · รวมกันแถวได้ 1.0

ข้อสังเกต: Built-up P(→ Built-up) = 0.97 = "sticky" — เมืองเป็นเมืองตลอด Cropland P(→ Built-up) = 0.13 = ทุก 10 ปีมี 13% ของ cropland กลายเป็นเมือง

🤔 Discussion: จำลองไป 20 ปี

ถ้า cropland → built-up ทุก 10 ปี 13%
แล้วในอีก 20 ปี cropland เหลือกี่ %?
A. ~70%
B. ~60%
C. ~50%
คำตอบ: ~60%
Step 1: 100% × (1 - 0.13) = 87%
Step 2: 87% × (1 - 0.13) = 75.7%
แต่ต้องคิดด้วยว่า class อื่นก็เปลี่ยนเป็น cropland เข้ามาบ้าง
→ ในทางปฏิบัติประมาณ ~60-70%
